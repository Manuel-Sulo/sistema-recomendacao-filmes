## PARTE 2 — DECISÕES TÉCNICAS FUNDAMENTAIS

### 2.1 APIs Externas — Análise e Escolha

| API | Plano Gratuito | Auth | Limite Free | Dados | Recomendação |
|---|---|---|---|---|---|
| **TMDB** | ✅ Sim | API key gratuita | Sem limite prático (fair use) | Filmes, séries, elenco, trailers, géneros, trending, similar, search, imagens | **⭐ Primária** |
| OMDB | ✅ Sim | API key gratuita | 1.000 req/dia | Dados básicos + ratings IMDb/RT | Complementar (ratings) |
| YouTube Data API v3 | ✅ Sim | API key | 10.000 units/dia | Trailers embed | Complementar (trailers) |
| Streaming Availability (watchmode) | ✅ Plano free | API key | 1.000 req/mês | Onde assistir (Netflix, etc.) | Opcional — limite apertado |

#### Decisão:
- **TMDB** → fonte principal e única necessária. Cobre: pesquisa, trending, detalhes, géneros, elenco, imagens, trailers (via `videos`), filmes similares/recomendados, ratings, discover por filtros
- **OMDB** → apenas para enriquecer com ratings IMDb e Rotten Tomatoes onde relevante
- Trailers: TMDB devolve chaves YouTube → embed direto sem precisar da YouTube API

> O backend PHP actua como proxy: guarda API keys, faz cache dos resultados, respeita rate limits. O Angular nunca chama TMDB directamente.

### 2.2 Web Scraping — Decisão

**Não será utilizado.** TMDB cobre 100% dos dados necessários via API oficial, gratuita e sem restrições de ToS. Scraping de IMDb, Letterboxd ou RT violaria os seus ToS e é tecnicamente frágil.

### 2.3 IA no Sistema — Decisão

O Lab menciona IA como ferramenta de desenvolvimento, não funcionalidade obrigatória. Porém, num sistema de **recomendação**, a IA é o próprio núcleo do problema. Implementar um motor de recomendação real é o que diferencia este sistema de um simples catálogo.

#### Motor de Recomendação — Abordagem Híbrida em PHP

| Estratégia | Descrição | Implementação |
|---|---|---|
| **Content-Based Filtering** | Recomendar filmes com géneros/elenco/diretor similares ao que o utilizador avaliou positivamente | PHP: calcular similaridade por géneros (Jaccard similarity) |
| **Collaborative Filtering simplificado** | "Utilizadores que avaliaram X também gostaram de Y" | PHP: query SQL com correlação entre avaliações |
| **Popularity-weighted** | Boost para filmes populares dentro dos géneros preferidos | Peso TMDB `popularity` + `vote_average` |
| **TMDB Similar/Recommended** | Endpoint nativo TMDB `/movie/{id}/similar` | Enriquecimento imediato sem cálculo |

> Abordagem **híbrida**: Content-Based (baseado no perfil do utilizador construído a partir das suas avaliações) + TMDB Similar como fallback para utilizadores novos (cold start). Tudo calculado em PHP com dados da BD local — sem dependência de serviços de ML externos.

**Cold Start** (utilizador sem histórico): mostrar trending TMDB + solicitar géneros favoritos no onboarding.

### 3.2 Base de Dados — Schema Completo

**7 tabelas**, normalizadas em 3NF:

```sql
-- ============================================================
-- SCHEMA: movie_recommender
-- ============================================================

CREATE DATABASE IF NOT EXISTS movie_recommender
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE movie_recommender;

-- ------------------------------------------------------------
-- T1: users
-- ------------------------------------------------------------
CREATE TABLE users (
    id             INT          PRIMARY KEY AUTO_INCREMENT,
    name           VARCHAR(100) NOT NULL,
    email          VARCHAR(150) UNIQUE NOT NULL,
    password_hash  VARCHAR(255) NOT NULL,
    role           ENUM('admin','user') NOT NULL DEFAULT 'user',
    language       VARCHAR(5)   NOT NULL DEFAULT 'pt',
    theme          VARCHAR(10)  NOT NULL DEFAULT 'dark',  -- cinema → dark default
    onboarded      TINYINT(1)   NOT NULL DEFAULT 0,       -- passou pelo onboarding?
    is_active      TINYINT(1)   NOT NULL DEFAULT 1,
    created_at     TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at     TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP
                                ON UPDATE CURRENT_TIMESTAMP
);

-- ------------------------------------------------------------
-- T2: password_reset_tokens
-- ------------------------------------------------------------
CREATE TABLE password_reset_tokens (
    id         INT          PRIMARY KEY AUTO_INCREMENT,
    user_id    INT          NOT NULL,
    token      VARCHAR(255) UNIQUE NOT NULL,
    expires_at TIMESTAMP    NOT NULL,
    used       TINYINT(1)   NOT NULL DEFAULT 0,
    created_at TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ------------------------------------------------------------
-- T3: user_genre_preferences
-- Géneros escolhidos no onboarding + inferidos via avaliações.
-- genre_id e genre_name vêm do TMDB (ex: 28 = Action).
-- ------------------------------------------------------------
CREATE TABLE user_genre_preferences (
    id         INT          PRIMARY KEY AUTO_INCREMENT,
    user_id    INT          NOT NULL,
    genre_id   INT          NOT NULL,   -- TMDB genre_id
    genre_name VARCHAR(50)  NOT NULL,
    source     ENUM('manual','inferred') NOT NULL DEFAULT 'manual',
    weight     DECIMAL(5,2) NOT NULL DEFAULT 1.00, -- aumenta com avaliações positivas
    created_at TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uq_user_genre (user_id, genre_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ------------------------------------------------------------
-- T4: ratings
-- Avaliação de 1-5 estrelas + review opcional.
-- Dados do filme guardados localmente (imutáveis pós-avaliação).
-- ------------------------------------------------------------
CREATE TABLE ratings (
    id           INT              PRIMARY KEY AUTO_INCREMENT,
    user_id      INT              NOT NULL,
    tmdb_id      INT              NOT NULL,   -- ID do filme no TMDB
    movie_title  VARCHAR(255)     NOT NULL,
    poster_path  VARCHAR(255),               -- path relativo TMDB
    release_year YEAR,
    rating       TINYINT UNSIGNED NOT NULL CHECK (rating BETWEEN 1 AND 5),
    review       TEXT,
    rated_at     TIMESTAMP        NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   TIMESTAMP        NOT NULL DEFAULT CURRENT_TIMESTAMP
                                  ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uq_user_movie (user_id, tmdb_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ------------------------------------------------------------
-- T5: watchlist  ("ver mais tarde")
-- ------------------------------------------------------------
CREATE TABLE watchlist (
    id           INT          PRIMARY KEY AUTO_INCREMENT,
    user_id      INT          NOT NULL,
    tmdb_id      INT          NOT NULL,
    movie_title  VARCHAR(255) NOT NULL,
    poster_path  VARCHAR(255),
    release_year YEAR,
    added_at     TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uq_user_movie_wl (user_id, tmdb_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ------------------------------------------------------------
-- T6: favorites
-- ------------------------------------------------------------
CREATE TABLE favorites (
    id           INT          PRIMARY KEY AUTO_INCREMENT,
    user_id      INT          NOT NULL,
    tmdb_id      INT          NOT NULL,
    movie_title  VARCHAR(255) NOT NULL,
    poster_path  VARCHAR(255),
    release_year YEAR,
    added_at     TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uq_user_movie_fav (user_id, tmdb_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ------------------------------------------------------------
-- T7: watch_history  ("já vi")
-- ------------------------------------------------------------
CREATE TABLE watch_history (
    id           INT          PRIMARY KEY AUTO_INCREMENT,
    user_id      INT          NOT NULL,
    tmdb_id      INT          NOT NULL,
    movie_title  VARCHAR(255) NOT NULL,
    poster_path  VARCHAR(255),
    release_year YEAR,
    watched_at   TIMESTAMP    NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uq_user_movie_hist (user_id, tmdb_id),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ------------------------------------------------------------
-- SEED: Admin padrão
-- ------------------------------------------------------------
INSERT INTO users (name, email, password_hash, role, onboarded)
VALUES ('Administrador', 'admin@movierecommender.ao',
        '$2y$12$...hash_aqui...', 'admin', 1);
```

**Relacionamentos:**
```
users (1) ──< password_reset_tokens  (N)
users (1) ──< user_genre_preferences (N)
users (1) ──< ratings                (N)
users (1) ──< watchlist              (N)
users (1) ──< favorites              (N)
users (1) ──< watch_history          (N)
```

**Nota de design:** `tmdb_id`, `movie_title`, `poster_path`, `release_year` são desnormalizados intencionalmente nas tabelas de utilizador pelas mesmas razões do sistema anterior — dados de identidade do filme são imutáveis no momento do registo, e renderizar listas sem chamar a API é um ganho de performance crítico. ### 3.4 Motor de Recomendação — Detalhe Técnico

```php
// RecommendationService.php

class RecommendationService {

    // 1. Construir perfil do utilizador
    // a) Géneros com peso: manual (onboarding) + inferred (via avaliações ≥ 4★)
    // b) tmdb_ids já vistos/avaliados → excluir das recomendações

    // 2. Content-Based Filtering
    // Para cada género favorito com peso > 0.5:
    //   → TMDB /discover/movie?with_genres={id}&sort_by=vote_average.desc
    //   → filtrar os já vistos/avaliados/em watchlist
    //   → pontuar: weight_género × vote_average_tmdb × popularidade_normalizada

    // 3. Collaborative Filtering simplificado (BD local)
    // Query:
    //   "Utilizadores que avaliaram com ≥4★ os mesmos filmes que eu,
    //    que outros filmes avaliaram com ≥4★ que eu ainda não vi?"
    // SQL:
    SELECT DISTINCT r2.tmdb_id, r2.movie_title, COUNT(*) as overlap_score
    FROM ratings r1
    JOIN ratings r2 ON r1.user_id != r2.user_id
                    AND r1.tmdb_id = (SELECT tmdb_id FROM ratings WHERE user_id = :uid)
                    AND r1.rating >= 4 AND r2.rating >= 4
    WHERE r2.tmdb_id NOT IN (SELECT tmdb_id FROM watch_history WHERE user_id = :uid)
      AND r2.tmdb_id NOT IN (SELECT tmdb_id FROM ratings WHERE user_id = :uid)
    GROUP BY r2.tmdb_id
    ORDER BY overlap_score DESC
    LIMIT 20;

    // 4. TMDB Similar (cold start / enriquecimento)
    // Para os últimos 3 filmes avaliados com ≥4★:
    //   → GET /movie/{id}/similar → adicionar ao pool filtrado

    // 5. Merge + deduplicação + ranking final
    // Score final = (content_score × 0.5) + (collaborative_score × 0.3) + (tmdb_similar × 0.2)
    // Ordenar por score desc, devolver top 20

    // 6. Atualizar user_genre_preferences (inferred)
    // Após avaliação ≥4★: incrementar weight do género em +0.1 (cap 2.0)
    // Após avaliação ≤2★: decrementar weight em -0.05 (floor 0.0)
}
```

### 3.6 Separação de Responsabilidades

**Backend (PHP puro):**

| Camada | Responsabilidade | Exemplo concreto |
|---|---|---|
| `index.php` | Bootstrap: config, cors, router | Entry point único |
| `Router` | METHOD + URI → Controller | `GET /api/movies/trending → MovieController::getTrending` |
| `Controller` | Valida input, chama Service, devolve JSON | Sem lógica de negócio |
| `Service` | Orquestração, cálculos, chamadas TMDB/OMDB | `RecommendationService::getForUser($userId)` |
| `Model` | Apenas queries PDO parametrizadas | `RatingModel::findByUser($userId)` |
| `Middleware` | Auth JWT, admin check | Cross-cutting antes do controller |
| `CacheHelper` | Cache ficheiro das respostas TMDB (TTL 1h) | Evita rate limit e melhora performance |
| `SimilarityHelper` | Algoritmos Jaccard, scoring, merge | Funções puras sem dependências |

**Frontend (Angular):**

| Camada | Responsabilidade |
|---|---|
| `Component` | Template + interação UI. Zero lógica de negócio |
| `Service` | HTTP calls, BehaviorSubject para estado partilhado |
| `Guard` | Auth e Admin route protection |
| `Interceptor` | Token injection, error handling global (401 → logout) |
| `Model/Interface` | Contratos TypeScript mapeados do JSON do backend |
| `Pipe` | `tmdb-image`, `runtime`, `year`, `translate` — formatação reutilizável |
| `SharedModule` | `movie-card`, `movie-grid`, `star-rating`, `trailer-modal` — zero lógica |

// ExportService.php — exemplo PDF de avaliações
public function exportRatingsPDF(int $userId): void {
    $ratings = $this->ratingModel->findByUser($userId);
    $pdf = new FPDF('P', 'mm', 'A4');
    $pdf->AddPage();
    $pdf->SetFont('Arial', 'B', 16);
    $pdf->Cell(0, 10, 'As minhas avaliações', 0, 1, 'C');
    foreach ($ratings as $r) {
        $pdf->SetFont('Arial', '', 12);
        $pdf->Cell(0, 8, $r['movie_title'] . ' (' . $r['release_year'] . ') - ' . $r['rating'] . '/5', 0, 1);
    }
    $pdf->Output('D', 'avaliacoes_' . date('Y-m-d') . '.pdf');
}
